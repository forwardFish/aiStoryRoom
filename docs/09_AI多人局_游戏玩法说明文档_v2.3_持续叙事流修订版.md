> 适用产品：AI多人局 Web MVP  
> 核心玩法：持续叙事时间线驱动的动态决策游戏  
> MVP 示例故事局：《桑田诏：嘉靖财政危局》

# 0. 文档目的
## 0.0A v2.2 主动谋划系统修订

### v2.3 持续叙事流修订

主游戏中间区域正式改为持续叙事时间线：

```text
故事
→ 决策建议
→ 玩家完整决定
→ AI 正在推演局势
→ 新故事
→ 本次变化
→ 下一次决策
```

不再把“消息卡片列表 + 固定决策区”作为默认体验。

前台 A/B/C 只展示行动标题；收益、风险、成功率和状态预测全部隐藏。

关键事件回应是当日两次主线决策中的一种呈现方式，不额外增加主线次数。

本版新增两套互补玩法：

```text
中间：主线决策——回应当前危机
右侧：主动谋划——主动交谈、调查、使用筹码、提前布局
```

统一术语：

```text
谋划
谋划机会
自拟谋划
执行谋划
```

禁止使用 `AP / 行动力 / 行动点 / 筹谋`。

## 0.1 MVP 固定节奏（v2.1 新增）

当前《桑田诏》MVP 不再使用“每天 1—3 次”的动态范围，而是固定：

```text
第 1—6 天：每天 2 次关键决策
第 1—6 天：每天 2 次谋划机会，可选使用
第 7 天：最终裁决，不新增普通决策，也不提供谋划机会
整局共 12 次关键决策，主动谋划理论上限 12 次
```

该固定节奏用于保证分支可测、成本可控和用户数据可比较。其他故事局未来可以配置不同数量，但不影响当前 MVP。



本文档回答一个问题：

> **这个游戏到底怎么玩？**

它不是技术 PRD，也不是剧本正文，而是定义玩家进入游戏后，从第一眼看到什么，到每一次如何做决策，再到局势如何变化、对手如何收到消息、最后如何结局的完整玩法规则。

本产品不是单个游戏，而是一个可以承载多个“故事局”的平台。  
因此，本玩法文档必须同时满足：

1. 能解释《桑田诏：嘉靖财政危局》怎么玩；
2. 能作为后续商战局、职场局、悬疑局、历史局的通用玩法模板；
3. 能指导前端页面、中间持续叙事时间线、AI 推演、状态反馈的开发。

# 1. 一句话玩法

> **玩家进入一个正在变化的局，在有限天数内阅读剧情消息，针对关键消息做决策，AI 根据决策推演后果，改变世界状态、人物关系和最终命运。**

更直白地说：

```text
别人做了事
↓
AI 把它变成一条剧情消息推到你面前
↓
你判断这件事会不会影响你
↓
你选择回应、利用、压制、交易、放任，或者自定义主线决策
↓
AI 推演后果
↓
你还可以在右侧主动谋划，交谈、调查、使用筹码或自拟布局
↓
主线与谋划结果共同改变局势
↓
你的行动又变成别人的剧情压力
↓
剧情继续往最终裁决推进
```

# 2. 玩家不是在玩什么

本产品不是：

```text
AI 聊天室
AI 写小说工具
普通互动小说
多人接龙写作
传统剧本杀
传统回合制桌游
纯策略数值游戏
```

玩家也不是在：

```text
写一段小说
和 AI 闲聊
自由扮演无限发散
猜固定真相
等所有人轮流提交行动
```

---

# 3. 玩家真正玩的是什么

玩家真正玩的，是：

```text
我是谁？
我现在处在什么局里？
别人刚刚做了什么？
这件事会不会影响我的命运？
我手里有什么权力、资源、情报和筹码？
我要不要插手？
我该怎么插手？
我的选择会改变谁？
我会升官发财、保住自己，还是被清算出局？
```

这个游戏的核心不是“剧情有多长”，而是：

> **每一条剧情消息都给玩家一个判断机会。**

# 4. 核心玩法结构

## 4.1 固定天数制

游戏不叫“第几轮”，而是按天推进。

例如标准局：

```text
第 1 天
第 2 天
第 3 天
……
第 7 天最终裁决
```

《桑田诏：嘉靖财政危局》MVP 推荐：

```text
第 1 天：改桑令下
第 2 天：地方执行
第 3 天：粮价上涨
第 4 天：暗账浮出
第 5 天：互相弹劾
第 6 天：京师回批
第 7 天：御前裁决
```

天数的作用：

```text
1. 限制游戏长度
2. 防止无限聊天
3. 给玩家时间压力
4. 让每一天都有明确主题
5. 让最终结局有收束点
```

---

## 4.2 每一天的基本流程

每一天在同一条中间叙事时间线中完成：

```text
1. 今日开场故事
2. 故事推进到第一个主线决策
3. AI 给出 3 个建议，玩家确认自己的决定
4. AI 正在推演局势
5. 新故事与本次变化自动出现
6. 玩家可从右侧发起主动谋划
7. 谋划结果继续进入中间时间线
8. 第二个主线决策
9. 日终故事
10. 进入下一天
```

一局完整循环：

```text
故事发生
↓
玩家选择建议或自拟决定
↓
规则校验
↓
AI 推演
↓
故事继续
↓
可见变化出现
↓
其他角色可能影响玩家
↓
必要时触发关键事件弹窗
↓
玩家继续回应或主动谋划
↓
最终裁决
```
## 4.3 每天消息数量限制

为了避免剧情发散，MVP 建议：

```text
每一天 3-5 条核心剧情消息
第 1—6 天每一天固定 2 次关键决策
第 1—6 天每一天提供 2 次谋划机会，可选且不结转
每一天必须进入日终结算
第 7 天必须进入最终裁决
```

每条消息都必须推动剧情。

不能出现无意义闲聊。

# 5. 主游戏页面怎么玩

主游戏只有一个页面：

```text
/game?runId=<runId>
```

8 个 UI 状态都发生在这个页面里。

页面结构：

```text
┌──────────────────────────────────────────────────────┐
│ 顶部：剧本 / 日期 / 地点 / 裁决倒计时 / 世界状态       │
├─────────────┬────────────────────────┬───────────────┤
│ 左侧：我是谁 │ 中间：持续叙事时间线      │ 右侧：主动谋划 │
│ 目标/资源/风险│ 故事/决策/推演/变化/回应 │ 人物/调查/筹码 │
└─────────────┴────────────────────────┴───────────────┘
```

## 5.1 左侧：我手里的牌

左侧固定展示：

```text
我的身份
命运问题
当前目标
我的资源
我的筹码
当前风险
```

左侧只回答：

> 我是谁？我有什么？我怕什么？

## 5.2 中间：真正的游戏现场

中间不是日志、聊天框或分页小说。

中间依次出现：

```text
故事正文
AI 建议
玩家输入
AI 推演
结果故事
本次变化
新压力
```

内容持续向下生长，玩家不离开主页面。

## 5.3 右侧：我主动做什么

右侧固定展示：

```text
今日谋划次数
可接触人物
正在推进
可用筹码
自拟谋划
进入下一天
```

右侧只是入口。

点击人物、调查、筹码或自拟谋划后，具体内容进入中间区域，仍使用同一套“建议 + 输入 + 提交”结构。

## 5.4 8 个 UI 状态

```text
UI 01 角色专属开场
UI 02 主线故事与决策
UI 03 AI 正在推演
UI 04 推演结果故事与变化
UI 05 局势记录
UI 06 关键事件弹窗
UI 07 他人影响故事与回应
UI 08 主动谋划
```

这些是同一页面的状态，不是 8 个路由。
# 6. 持续叙事时间线怎么玩

## 6.1 故事不是系统日志

错误：

```text
浙江巡抚选择了【越级上奏】。
```

正确：

```text
驿站快马离开杭州府。

有人看见浙江巡抚的幕僚亲自护送一封急奏北上。
奏中只写改桑进度，不提粮价与民怨。
```

对方的行动必须被转译成当前角色可以感知的故事。

## 6.2 时间线中的内容

### 1）开场故事

交代日期、地点、当前压力和角色处境。

### 2）角色行动故事

把 AI 角色或未来真人玩家的行动转成当前角色视角。

### 3）私密情报故事

只展示当前玩家应该知道的信息。

### 4）决策模块

故事自然推进到必须判断的位置后，紧接出现：

```text
你要如何应对？

A 截留奏疏
B 追加密奏
C 放任巡抚

[输入你的决定……]

提交决策
```

### 5）推演状态

```text
你的决定

追加密奏，并让县令整理粮价证据。

AI 正在推演局势……
```

### 6）结果故事

先讲清楚发生了什么，不立刻堆数值。

### 7）本次变化

紧接结果故事自动出现：

```text
本次变化

皇帝信任 +4
巡抚敌意 +3
获得情报：京中注意到两份奏报差异
```

### 8）日终与结局

同样作为故事自然追加，不跳独立页面。

## 6.3 连续阅读规则

取消：

```text
继续阅读
下一页
查看变化
返回局势
完成
下一步
```

玩家只在需要作出行动时点击提交。

## 6.4 局势记录

局势记录默认隐藏。

打开后只覆盖中间区域，展示：

```text
我的决策
他人影响
世界变化
数值变化
```

它只负责回看，不负责决策。关闭后恢复原来的滚动位置。

## 6.5 关键事件

当其他角色的行动直接影响玩家时，可以弹出：

```text
重要事件发生

巡抚对你的怀疑正在加深

立即处理
稍后处理
```

普通世界变化不能弹窗。

点击立即处理后，中间展示角色视角故事和回应建议；稍后处理必须保留待处理状态。
# 7. 玩家如何做决策

## 7.1 三个建议 + 一个输入框

每个关键 prompt 只展示：

```text
A 行动标题
B 行动标题
C 行动标题
```

不展示：

```text
说明
收益
风险
成功率
数值预测
```

下方始终显示统一输入框：

```text
你也可以写下自己的决定：

[输入你的处理方式……]
0 / 200

提交决策
```

## 7.2 点击建议

玩家点击建议后，标题自动写入输入框。

例如：

```text
追加密奏
```

玩家可以直接提交，也可以补充：

```text
追加密奏，并让县令整理粮价证据作为附件。
```

## 7.3 完全自定义

玩家可以不选任何建议，直接输入自己的处理方式。

输入最大 200 字。

AI 需要检查：

```text
角色身份
时代背景
当前资源
权力边界
是否直接宣布结果
是否操控其他角色
是否跳过剧情
```

## 7.4 三种提交文案

同一组件根据场景显示：

```text
提交决策
提交回应
提交谋划
```

关键事件回应在 MVP 中仍占用当日 2 次主线决策中的一个槽位。
# 8. 主动谋划系统

## 8.1 为什么需要主动谋划

如果玩家只能等待中间出现 A/B/C，产品会退化成互动小说。

主动谋划让玩家获得：

```text
应对权：处理眼前危机
主动权：提前调查、接触人物、使用筹码和布置后手
```

## 8.2 谋划机会

```text
第 1—6 天每天 2 次
第 7 天 0 次
成功执行后使用 1 次
被 ActionGuard 拒绝不使用
未使用不结转
完成两次主线决策即可进入下一天
```

页面显示：

```text
今日谋划 2 / 2
● ●
```

不使用 AP、行动力、行动点或筹谋。

## 8.3 四类主动谋划

### 人物交谈

点击人物后选择目的：

```text
试探态度
索取情报
施加压力
提出交易
自拟说辞
```

### 派遣调查

通过幕僚或可用人员调查当前局势：

```text
查驿站
查田契
查粮仓
查商会往来
保护或接触证人
```

### 使用筹码

筹码包括：

```text
证据
线索
秘密
人情
承诺
权力
关系
特殊情报
```

玩家必须选择使用对象、使用目的和公开程度。

### 自拟谋划

输入不超过 200 字：

```text
派幕僚暗查驿站登记，确认巡抚急奏由谁经手。
```

按钮：

```text
执行谋划
```

## 8.4 谋划结果

谋划结果不能只写“成功”。

提交后：

```text
AI 正在推演局势……
```

随后中间时间线自动追加：

```text
结果故事
本次变化
```

本次变化可包括：

```text
获得线索
关系变化
风险变化
目标进度
筹码状态
留下痕迹
```

前台不提前展示收益和风险预测。
## 8.5 主线决策与主动谋划

| 对比项 | 主线决策 | 主动谋划 |
|---|---|---|
| 触发 | 世界提出压力 | 玩家主动出手 |
| 必须性 | 必须 | 可选 |
| 每日数量 | 2 次 | 最多 2 次 |
| 影响重点 | 主线局势 | 线索、关系、筹码、任务、伏笔 |
| 是否推进天数 | 两次完成后可推进 | 不能替代主线 |
| 结果展示 | 中间叙事时间线 | 中间叙事时间线 |

一句话：

> **中间决定你如何回应眼前危机；右侧决定你是否提前布局下一场危机。**

# 9. 决策后会发生什么

玩家提交主线决策、关键事件回应或主动谋划后，系统执行：

```text
1. 记录玩家最终提交文本
2. 进入极简 AI 推演状态
3. 规则引擎确定合法状态变化
4. AI 生成结果故事
5. 公共投影筛选玩家可见变化
6. 新故事和本次变化自动追加
7. 必要时生成角色反应或关键事件
8. 更新右侧可用人物、目标和筹码
```

玩家不会看到：

```text
系统判断步骤
概率
权重
其他角色隐藏数值
内部因果链
```

玩家会看到：

```text
发生了什么
自己得到了什么信息
哪些可见状态发生变化
下一步是否需要回应
```
# 10. AI 角色怎么玩

## 10.1 AI 角色不是主动玩家

AI 角色不是为了“赢”而主动谋划。  
它们是局势参与者。

AI 角色只在三种情况下行动：

```text
1. 玩家影响到它
2. 主线事件推着它
3. 它的风险或利益达到阈值
```

---

## 10.2 AI 角色触发示例

```text
粮价 > 70
→ 清流县令请求开仓

皇帝疑心 > 75
→ 司礼监入浙探查

巡抚声望被压制
→ 巡抚越级上奏

商会清算风险过高
→ 商会投靠司礼监

县令信任过低
→ 县令保留证据副本，不再完全信任你
```

---

## 10.3 AI 角色有默认惯性

如果真人玩家不在线，角色按默认倾向推进。

例如：

```text
浙江巡抚：继续推进改桑，并保持向内阁报功倾向
清流县令：继续保护百姓，并暗查田契
江南商会：优先保护商会利益，寻找强权庇护
司礼监：优先探查皇帝真正关心的银路
内阁财政派：优先寻找能补银或能背锅的人
```

这样游戏不会被某个玩家卡死。

# 11. 单人模式怎么玩

MVP 先做单人。

玩家控制：

```text
浙江总督
```

AI 扮演：

```text
浙江巡抚
清流县令
江南商会
司礼监
内阁财政派
皇帝
```

单人模式流程：

```text
进入故事局
↓
确认角色：浙江总督
↓
第 1 天开始
↓
系统生成剧情消息
↓
AI 角色根据局势反应
↓
玩家针对关键消息决策
↓
AI 推演后果
↓
进入下一天
↓
第 7 天最终裁决
```

单人模式的关键体验：

```text
不是我一个人在读故事；
而是我面对一群有立场、有利益、有默认反应的 AI 角色。
```

# 12. 多人模式怎么玩

多人模式不要求同步。

例如：

```text
玩家 A：浙江总督
玩家 B：浙江巡抚
玩家 C：清流县令
玩家 D：江南商会
```

玩家 B 做出决策：

```text
越级上奏，抢先向内阁报功。
```

系统不会把这个显示成系统日志，而是把它转译成总督视角剧情消息：

```text
【第 3 天 · 午后】

驿站传来消息，巡抚府有急奏北上。

奏中称浙江改桑已有成效，地方民情可控，若朝廷嘉奖及时，十日内便可见银。

你的幕僚提醒：

“大人，巡抚这是要先一步坐实功劳。若日后出事，他也可说自己早已尽力，是总督府压慢了局势。”
```

总督玩家回应后，也会反过来影响巡抚玩家的持续叙事时间线。

多人模式的本质：

> **每个玩家都在自己的持续叙事时间线里做判断，但彼此的选择会互相转译成压力。**

---

# 13. 如何避免无限聊天

本产品必须严格避免无限聊天。

## 13.1 固定天数

每局固定：

```text
3 天 / 7 天 / 10 天
```

MVP 使用 7 天制。

---

## 13.2 每天消息上限

每一天最多：

```text
3-5 条核心剧情消息
第 1—6 天固定 2 次关键决策
第 1—6 天最多 2 次主动谋划
1 次日终回响
第 7 天只进行最终裁决
```

---

## 13.3 每次决策后最多生成 3 类结果

```text
1 条自己的结果消息
1 条影响他人的剧情消息
1 条隐藏暗线变化
```

---

## 13.4 每条消息必须服务主线

一条消息必须至少带来以下之一：

```text
新压力
新机会
新风险
新关系变化
新决策点
新结局影响
```

否则不进入主叙事时间线。

---

## 13.5 每天必须日终结算

每天不能无限决策。

完成当天两次主线决策后，可以继续使用剩余谋划机会，也可以直接进入：

```text
日终回响
```

日终回响用于：

```text
总结今日局势
展示今日关键决策
更新世界状态
提示明日风险
推动进入下一天
```

# 14. 三层故事线

为了让游戏不散，所有剧情都应该落在三层线上。

## 14.1 个人线

每个角色自己的目标、秘密、风险和命运。

```text
我是谁？
我想要什么？
我害怕什么？
我知道什么秘密？
我现在面临什么选择？
```

## 14.2 关系线

角色之间的信任、敌意、交易、把柄和背叛风险。

```text
我信任谁？
谁在隐瞒我？
我掌握了谁需要的信息？
谁的行动改变了我的处境？
```

## 14.3 世界线

整个局的大事件和最终压力。

```text
国库缺银
皇帝催银
浙江改桑
粮价上涨
官商勾结
地方民怨
最终裁决
```

好的剧情推进应该是：

```text
个人线推动关系线
关系线反过来改变世界线
世界线再压迫每个人做选择
```

---

# 15. 三个回响

三个回响继续作为后台推演要求，但不再做成独立因果卡。

每次关键行动后，系统内部仍需生成：

```text
个人回响
他人回响
世界回响
```

前台表达方式：

```text
个人回响 → 融入结果故事和自己的状态变化
他人回响 → 融入角色反应或后续关键事件
世界回响 → 融入世界故事和顶部状态变化
```

玩家需要理解因果时，可以打开局势记录回看，但不显示完整后台因果链。
# 16. 结局怎么玩

第 7 天进入最终裁决。

AI 根据：

```text
世界变量
人物关系
关键决策
把柄流向
隐藏暗线
谁得利
谁背锅
谁保命
谁被清算
```

生成两类结局。

## 16.1 全局结局

示例：

```text
本局结局：商人救国，内廷得利

改桑没有彻底失败。
国库得到第一笔银子。
但江南丝路从此被商会与司礼监共同控制。
内阁保住体面，浙江民心留下裂痕。
```

## 16.2 个人结局

示例：

```text
你的最终下场：小胜

你稳住了浙江，也没有让改桑彻底失败。
皇帝认为你尚可用，但不可不防。
你被调任东南军务重臣，名义升迁，实际被内阁与司礼监共同牵制。

关键三手：
1. 追加密奏，保留解释权
2. 逼商会平粮，暂稳民心
3. 放任巡抚抢功，最后借暗账反制

命运债：
你利用了县令，却没有真正保住他。
```

# 17. 《桑田诏》完整一日玩法示例

## 第 3 天 · 午后

### 故事

```text
驿站快马离开杭州府。

有人看见浙江巡抚的幕僚亲自护送一封急奏北上。
奏中称浙江改桑已有成效，却没有提粮价，也没有提三县拒签田契。

你的幕僚低声提醒：

“大人，若这封奏疏先到内阁，巡抚便是功臣；
若之后民怨爆发，您这个总督就是压不住局的人。”
```

### 决策

```text
你要如何应对？

A 截留奏疏
B 追加密奏
C 放任巡抚

[输入你的决定……]

提交决策
```

玩家点击 B，输入框自动出现：

```text
追加密奏
```

玩家补充为：

```text
追加密奏，并让县令整理粮价证据作为附件。
```

### 推演

```text
你的决定

追加密奏，并让县令整理粮价证据作为附件。

AI 正在推演局势……
```

### 新故事

```text
你没有截留巡抚奏疏。

当夜，总督府另有一封密奏从侧门送出。
奏中不直接指责巡抚，只写粮价、民情与不可躁进。

两份口径不同的浙江奏报，正先后奔向京师。
```

### 本次变化

```text
皇帝信任 +4
内阁疑心 +2
巡抚敌意 +3
获得情报：司礼监开始注意奏报差异
```

没有“查看变化”“下一步”或“完成”按钮。

若随后出现新压力，新的故事和决策继续追加到下方。
# 18. 多故事局如何复用这套玩法

本玩法不是只为《桑田诏》设计。

不同故事局只需要替换：

```text
世界背景
角色身份
核心冲突
天数主题
世界变量
关系变量
消息模板
决策选项
结局规则
```

底层循环不变：

```text
剧情消息
↓
主线决策
↓
主动谋划
↓
AI 与规则推演
↓
状态反馈
↓
新剧情消息
↓
最终结局
```

故事局：

```

核心变量：

```

投资人代表私下约见 CFO，希望提前查看真实现金流。

玩家决策：

```text
A 公开财务问题
B 先稳住投资人
C 推给销售团队解释

[输入你的处理方式……]
```

# 19. 玩家体验标准

一局结束后，玩家应该能说出：

```text
我做了几个关键决定，也主动进行过谋划。
主线决策和谋划改变了别人对我的态度。
有些事情当时看起来赢了，后来变成风险。
最后我的结局不是系统随机给的，而是我自己一步步推出来的。
```

如果玩家只觉得：

```text
我读了一堆 AI 写的剧情。
我只是在点 A/B/C，没有主动布局。
最后给我一个结局。
```

那玩法失败。

# 20. 最终总结

本游戏的本质是：

> **持续叙事时间线驱动的动态决策游戏。**

核心循环：

```text
其他角色的行动被转译成故事
↓
故事把压力推到玩家面前
↓
AI 给出三个可参考的行动方向
↓
玩家确认自己的完整决定
↓
AI 与规则引擎继续推演
↓
新故事和本次变化自动出现
↓
玩家还可以从右侧主动谋划
↓
所有人的行动共同把局推向最终裁决
```

最终目标不是让玩家读完一堆消息，而是让玩家产生：

> **我一直在故事里判断、回应和布局，并亲手把这个局推向了某种结局。**
