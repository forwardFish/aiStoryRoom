# AI多人局｜多人推演效果测试流：身份驱动版 v2.1

> 目的：把原稿从“事件触发型推演”改成“角色身份驱动型推演”。  
> 核心测试问题：多人推演是否像真人在局里博弈，而不是像系统替角色点按钮。

---

# 0. 重写原则

上一版的问题是：

```text
玩家做选择
↓
系统给状态变化
↓
其他角色收到影响
```

这套逻辑成立，但还不够真实。

这版改成：

```text
角色身份
↓
他知道什么 / 不知道什么
↓
他害怕什么 / 想得到什么
↓
他在这种场合最可能怎么判断
↓
他选择一个对自己最有利、但未必对全局最好的动作
↓
动作被转译成别人看到的剧情压力
↓
别人再根据自己的身份和私密目标反应
```

真正要模拟的不是“剧情按钮”，而是：

> 人在身份、权力、恐惧、利益、误判下的决策。

# 1. 测试局设定

故事名：

```text
《桑田诏：嘉靖财政危局》
```

嘉靖朝财政吃紧，朝廷要求浙江推进“改稻为桑”，以丝税补国库。

但浙江不是一张白纸。

这里同时压着几条线：

```text
国库缺银
海防欠饷
江南粮田
商会囤粮
巡抚抢功
县令护民
司礼监盯银路
内阁找责任人
```

所有人表面都在说“为国”，但每个人真正想保的是自己的命运。

# 2. 参与玩家身份模型

## 玩家 A：浙江总督，也就是“你”

公开身份：浙江军政大员，奉旨督办改稻为桑，同时负责海防、地方稳定和奏报京师。

真实目标：

```text
不能被定为拖延国策
不能让巡抚抢功坐大
不能让商会控制银路
不能让县令把民情直接炸到京师
不能让皇帝觉得自己欺瞒
最好让皇帝意识到：浙江不可无你
```

你在局里的真实心理：

```text
浙江不能乱。
但如果一定要有人背锅，不能是我。
如果能稳住浙江，还能借局升迁，那才是最好。
```

---

## 玩家 B：浙江巡抚

真实目标：

```text
抢在总督前面报功
把自己塑造成新政功臣
进入京师财政派视野
避免官商暗线暴露
```

他的真实心理：

```text
朝廷要的是银子，不是解释。
谁先把进度报上去，谁就是能办事的人。
总督越谨慎，越像拖延。
我必须先坐实功劳，否则功劳永远是总督的，责任才是我的。
```

---

```text
查清改桑背后的官商夺田
保住百姓不被商会低价吃田
让证据进入能真正起作用的人手里
同时保护自己不被上级牺牲
```

他的真实心理：

```text
我不能抗旨。
但如果我什么都不做，百姓的田就没了。
总督也许能用，但总督也是官场中人。
证据不能只交一份。
```

```text
官员缺银时第一个想到我。

所以我给银子可以，但必须换保护。
不能只靠一个官，要多边下注。
```

# 第 1 轮：第 1 天 · 巡抚请命

## 公开剧情消息

```text
【第 1 天 · 清晨 · 杭州总督府】

京师急诏抵达浙江。

诏书写得简洁：

江南择地改稻为桑，以桑养蚕，以丝换银，以银补国用。

你合上诏书，案上还有另一封海防军报。

军报写道：

沿海军饷已拖两月，若再不补给，军心难稳。

幕僚低声道：

“大人，朝廷要银，地方要粮，军中要饷。浙江这一局，不能只看桑田。”
```

## 巡抚为什么会出手

巡抚看到诏书的第一反应不是“民生”，而是“机会”。

他判断：

```text
总督不敢公开反对圣旨。
所以我可以用“奉诏急办”压他表态。
```

他决定：

```text
立即提交三县改桑计划。
```

表面说法：

```text
“大人，朝廷催银甚急，浙江不可迟疑。”
```

真实目的：

```text
先把自己和“执行新政”绑定。
未来如果民怨起，就说自己奉诏执行，是总督府节制不明。
```

## 给总督的待决策消息

```text
【浙江巡抚 · 请命】

午前，浙江巡抚入总督府请见。

他呈上一份初拟名册，称嘉兴、绍兴、湖州三地最宜先行改桑，并请你准许他立即督办。

他说：

“朝廷催银甚急，若浙江迟疑，京师必疑我等不奉诏。臣愿先担执行之责。”

你看着那份名册。

三地皆为熟田。

若立刻推进，改桑进度会上升，但粮价可能先动。
若压住巡抚，京师可能疑你拖延。

巡抚在等你的态度。
```

## 你的选项

```text
A. 准许巡抚先行推进
B. 要求先核田清册
C. 表面同意，私下查田契
D. 自定义决策
```

## 真实玩家式选择：浙江总督选择 C

你的真实想法：

```text
圣旨刚到，谁第一个说慢，谁就像在拖国策。
但巡抚这么快拿出三县名册，说明这份名册可能早就有人准备。
若背后有商会手脚，这就是未来压住巡抚的把柄。
```

所以你选择：

```text
表面同意巡抚筹办。
私下命幕僚查三县田契来源。
```

## 其他玩家的真实反馈

### 巡抚

他误判了你。

```text
总督果然不敢拦圣旨。
他只是例行提醒。
我必须趁这几天把进度坐实。
```

行动：

```text
立刻让幕僚誊写三县公文。
给内阁财政派准备一份“浙江已动”的预报口信。
```

### 县令

他没有收到总督明确保护，只看到巡抚府开始动。

```text
上面可能已经默许。
但我不能坐等百姓田契被吃掉。
```

行动：

```text
暂不公开老农田契。
暗中记录来问田价的商号名称。
让可信书吏抄一份田契副本。
```

### 商会

商会判断：

```text
巡抚能给速度，总督能给保护。
不能只下注一边。
```

行动：

```text
向巡抚府递“助改桑”银粮意向。
同时给总督府送拜帖。
```

## AI 推演结果

```text
总督表面同意，避免立刻和巡抚冲突。
但暗查田契让暗账线启动。

巡抚误判总督态度，以为总督只是谨慎，不敢真正阻拦。
县令开始独立留证，但还没有完全信任总督。
商会双线下注，未来可能根据谁更强转向谁。
```

# 第 2 轮：第 1 天 · 入夜商会递帖

这一轮测试“商会不是资源包，而是有算盘的人”。

## 商会为什么出手

商会会首不是单纯送钱。

他想的是：

```text
现在所有官都缺钱。
我给银子，就必须换未来的保护和特权。
但不能把话说死，否则将来出事就是证据。
```

所以他不会直接说“我要买田”。

他会说：

```text
商会愿为朝廷分忧。
```

## 总督收到的剧情消息

```text
【江南商会 · 递帖】

入夜前，江南商会会首送来拜帖。

帖子写得很客气：

“商会愿为朝廷分忧。若改桑确需先银后税，商会可先垫一笔粮银，用于安抚地方。”

但帖子最后一句话意味深长：

“商路若稳，国用自足。商路若疑，银水难流。”

幕僚低声道：

“大人，这是来要保护的。”
```

## 你的选项

```text
A. 公开召商会议粮银
B. 私下见商会，只听不许
C. 暂不见商会
D. 自定义决策
```

## 真实玩家式选择：浙江总督选择 B

你的真实想法：

```text
公开召见太容易被县令和清流抓住。
完全不见，又会失去商会银粮这个筹码。
```

你选择：

```text
私下见商会，只听不许。
不写承诺，不收银票，只问粮仓、丝路和三县田价。
```

## 商会的真实反馈

商会发现：

```text
总督愿意见我。
但不愿留下字据。
```

他的判断：

```text
总督比巡抚难缠。
巡抚要的是政绩，总督要的是可控。
但正因为他可控，他未来也许更能保护商会。
```

行动：

```text
暂不把全部银粮押给巡抚。
保留三成粮仓不动。
给总督一份模糊账目，只透露“可垫银”，不透露全部田契。
```

## 巡抚的真实反馈

巡抚听说商会入了总督府。

他的心理：

```text
商会原本该靠我推进改桑。
现在却去试探总督。
如果总督拿到商会银粮，我的功劳会被分走。
```

行动：

```text
加快三县公文下发。
让三县先把名册报上来，抢在总督与商会谈出结果前坐实进度。
```

## 县令的真实反馈

县令听说商会夜入总督府。

他的心理：

```text
总督不是没和商会接触。
那我交证据给总督，最后会不会被压下？
```

行动：

```text
继续查。
但不把全部材料放在一个地方。
```

## AI 推演结果

```text
总督获得商会试探性靠近，但没有被商会完全绑定。

商会认为总督值得下注，但仍保留底牌。
巡抚感到资源被总督分走，开始加快推进。
县令对总督信任没有崩，但开始有戒心。
```

## 状态变化

```text
商会依赖：38 → 45
总督权威：62 → 64
巡抚敌意：32 → 38
县令信任：50 → 48
国库银预期：30 → 35
清算风险：45 → 47
```

### 给巡抚

# 第 3 轮：第 2 天 · 县令密信与巡抚催政

这一轮测试“县令不是单纯交证据，而是在保护自己”。

## 公开剧情消息

```text
【第 2 天 · 清晨】

巡抚府昨夜发出三道公文，要求嘉兴、绍兴、湖州三地限期上报名册。

公文措辞极重：

“有迟误者，以误国论。”

杭州米价小涨。

粮铺掌柜都说只是雨水误船，但你知道消息传得太快了。

幕僚递上一句话：

“大人，巡抚已经开始往下压了。”
```

## 县令为什么递密信

县令现在不是“正义冲锋”。

他知道：

```text
直接公开，就是抗旨。
直接交总督，可能被压下。
什么都不做，百姓田会被吃掉。
```

所以他选择：

```text
递密信，但不署名。
给总督线索，但不把自己完全暴露。
保留副本，但不马上公开。
```

## 总督收到的剧情消息

```text
【私密 · 清流县令密信】

午后，一封未署名密信送入总督府。

信中写道：

“本县尚未丈量，已有商号执契来问桑田价。
百姓未卖田，田名似已在册。
若此事属实，改桑恐变夺田。”

信末只有一句：

“望大人明察。”

幕僚认出笔迹，是嘉兴一名清流县令。

你现在手里有了第一条真正的线索。
```

## 你的选项

```text
A. 保护县令，命其继续查账
B. 要求县令停止私查，只报民情
C. 密令县令查账，但证据必须先送总督府
D. 自定义决策
```

## 真实玩家式选择：浙江总督选择 C

你的真实想法：

```text
县令是刀。
刀能伤敌，也能伤自己。
我不能让他自由公开，也不能把他压下去。
```

你选择：

```text
给县令有限保护。
允许查账。
但证据必须先送总督府。
```

真实目的：

```text
不是立刻清弊。
而是掌握证据流向。
```

## 巡抚的真实反馈

巡抚听说嘉兴县令在查田契。

他的判断：

```text
县令不会无缘无故敢查。
背后可能有总督。
如果我现在公开压县令，等于承认自己怕查。
不如继续催名册，把进度做成事实。
```

行动：

```text
不公开冲突。
继续催三县名册。
同时让师爷提醒地方胥吏：旧账不要留在县衙。
```

## 县令的真实反馈

县令收到总督回信：

```text
可查。
但证据先送总督府。
```

他的心理：

```text
总督愿意用我。
但总督也想控制我。
```

行动：

```text
继续查。
送一部分证据给总督。
完整副本另行保存。
```

## 商会的真实反馈

商会发现县令在查田契。

他的心理：

```text
巡抚能推进，但未必能保护我。
总督在看账。
县令在查账。
我需要更高一层的保护。
```

行动：

```text
派人试探司礼监织造局。
不直接投靠，只先递“贡丝银路”的消息。
```

## AI 推演结果

```text
总督与县令形成有限合作。
巡抚继续推进，同时开始清理痕迹。
商会开始寻求内廷保护。
```

# 第 5 轮：第 3 天 · 商会控粮与县令信任裂痕

## 商会为什么要这个“护身符”

商会会首现在并不轻松。


他的判断：

```text
我必须用粮食换一个保护信号。
```

## 你的选项


```text

同时命幕僚记录粮价变化，不记录交易细节。
```

这是一个灰色选择。

你确实稳住了粮价。

商会会把“忠商寒心”四个字记下来。

县令会从粮价突然回落中察觉：商会和总督府达成了某种交换。

# 5. 这 5 轮为什么比上一版更真实

## 5.1 巡抚不是“反派”，而是有政治生存逻辑

他越级上奏不是因为系统让他“抢功”。

```text
县令在查账
```

所以他的行为合理。

因为：



所以很多选择都是灰色的。

# 6. 对产品设计的直接启发

## 6.1 每个 AI 玩家必须有“角色决策模型”

每个角色不应只写背景介绍，而要有一套决策模型。

建议字段：

```json
{
  "roleKey": "xunfu",
  "publicIdentity": "浙江巡抚",
  "publicGoal": "推进改桑，尽快见银",
  "realGoal": "抢在总督前报功，借新政入京",
  "fear": [
    "总督掌握暗账",
    "商会反咬",
    "县令公开证据"
  ],
  "decisionBias": [
    "倾向抢先进度",
    "倾向向内阁报功",
    "被压制时反咬总督",
    "暗账风险高时切割商会"
  ],
  "informationStyle": "只公开对自己有利的信息",
  "betrayalThreshold": {
    "merchantRisk": 65,
    "xunfuHostility": 70
  }
}
```

---

## 6.2 每轮 AI 推演必须先问“这个角色为什么这么做”

不要直接生成动作。

应该先生成内部判断：

```json
{
  "roleKey": "merchant",
  "privateReasoningSummary": "商会认为巡抚推进快但未必能保护自己，总督能稳局但不会给明面承诺，因此选择用放粮换模糊保护，同时保留司礼监通道。",
  "chosenAction": "放出三成平价粮，但保存总督府口头保护记录。",
  "surfaceMessage": "商会愿为朝廷分忧，先放粮稳市。",
  "hiddenIntent": "将来若被清算，可证明总督府也曾使用商会资源。"
}
```

这个 `privateReasoningSummary` 不一定给玩家看，但要给系统作为推演依据。

---

## 6.3 玩家页面应展示“我看到的”，后台保存“他真正想的”

玩家不应该看到：

```text
商会真实目的是将来反咬总督。
```

玩家应该看到：

```text
商会放出三成平价粮。
但账房临走前，特意问了一句：
“总督府今日这句话，是否可理解为商会是在为朝廷分忧？”
```

这就是信息不对称。
