# AI 多人局｜游戏玩法说明文档 v1.0

> 文档类型：玩法说明 / 游戏规则 / 交互体验说明  
> 适用范围：AI 多人局平台下的所有故事局  
> MVP 示例：《桑田诏：嘉靖财政危局》  
> 核心形态：剧情消息驱动的动态决策游戏  
> 目标读者：产品、设计、前端、后端、AI 工程、测试、内容策划

---

# 0. 文档目的

本文件回答一个核心问题：

> **这个游戏到底怎么玩？**

它不是某一个剧本的详细剧情文档，而是整个 **AI 多人局平台** 的通用玩法规则。

未来平台可以有很多故事局：

```text
《桑田诏：嘉靖财政危局》
《融资前夜：谁拿走了控制权》
《晋升名单公布前》
《九子夺嫡》
《董事会逼宫》
《午夜便利店》
```

不同故事局可以更换题材、角色、变量、剧情文本和美术风格，但底层玩法一致：

```text
剧情消息出现
↓
玩家判断局势
↓
玩家选择或自定义决策
↓
AI 推演后果
↓
世界状态、人物关系、风险和命运发生变化
↓
新剧情消息继续出现
↓
固定天数后进入最终结局
```

一句话：

> **玩家不是在读故事，而是在一个不断变化的局里做决策。**

---

# 1. 游戏一句话

> **玩家进入一个有限天数的 AI 推演局，扮演拥有身份、目标、秘密和筹码的角色，通过一条条剧情消息做决策，改变局势、影响别人，最终得到自己的命运结局。**

---

# 2. 这个游戏不是这些

本产品不是：

```text
AI 聊天室
AI 小说生成器
多人写作接龙
传统剧本杀
普通互动小说
纯策略数值游戏
开放世界 RPG
DND 跑团完整规则系统
```

它也不是简单的：

```text
读一段剧情 → 选 A/B/C → 看下一段剧情
```

而是：

```text
别人做的事会变成剧情压力推到你面前
你必须根据自己的目标、资源和风险做判断
你的判断会改变世界状态、人物关系和最终下场
```

# 3. 核心玩法定义

## 3.1 核心玩法名

建议内部统一称为：

> **剧情消息驱动的动态决策局**

也可以简称：

> **故事局**

## 3.2 核心体验

玩家真正玩的不是剧情选择题，而是以下判断：

```text
我是谁？
我现在想保命、升迁、发财、掌权，还是留名？
别人刚刚做了什么？
这件事会不会害到我？
我要不要插手？
我是公开回应，还是私下布局？
我要利用谁？
我要牺牲谁？
我要留下什么后手？
如果失败，我会不会背锅？
```

## 3.3 游戏核心循环

```text
每日开局
↓
剧情消息流推进局势
↓
关键消息触发玩家决策
↓
玩家选择 A/B/C 或自定义决策
↓
AI 判断决策是否合法
↓
AI 推演后果
↓
右侧状态变化
↓
如果影响别人，生成对方视角剧情消息
↓
进入下一条剧情消息
↓
日终回响
↓
进入下一天
↓
最终裁决
```

# 4. 游戏节奏：用“天数”限制，而不是无限聊天

## 4.1 不叫回合，叫天

游戏不使用“第 1 回合、第 2 回合”这种桌游式表达。

统一使用：

```text
第 1 天
第 2 天
第 3 天
……
第 7 天最终裁决
```

这样玩家会更有压迫感：

> **时间在流逝，局势每天都在逼近最终审判。**

## 4.2 标准局长度

推荐三种局长：

| 类型 | 天数 | 时长 | 适用场景 |
|---|---:|---:|---|
| 短局 | 3 天 | 10-20 分钟 | 新手试玩 / 快速体验 |
| 标准局 | 7 天 | 30-45 分钟 | MVP 主体验 |
| 长局 | 10 天 | 60-90 分钟 | 多人深度局 / 高级剧本 |

MVP 推荐：

```text
7 天制
```

## 4.3 每天的结构

每一天由 4 部分构成：

```text
1. 今日开局剧情
2. 剧情消息流
3. 玩家关键决策
4. 日终回响
```

## 4.4 每天消息数量限制

为了避免无限发散，每天限制：

```text
每天 3-5 条核心剧情消息
每天 1-3 次关键决策
每天必须进入日终回响
最后一天必须进入最终裁决
```

# 5. 主游戏页怎么玩

## 5.1 页面三栏结构

游戏主页面采用三栏结构：

```text
┌────────────────────────────────────────────────────────┐
│ 顶部：剧本名 / 第几天 / 地点 / 当前阶段 / 剩余天数       │
├──────────────┬──────────────────────────┬──────────────┤
│ 左侧信息栏     │ 中间主游戏区               │ 右侧结果栏      │
│ 我的身份       │ 剧情消息流                 │ 世界状态        │
│ 我的目标       │ 其他角色行动剧情化消息       │ 人物关系        │
│ 我的资源       │ 系统剧情推进               │ 当前风险        │
│ 我的筹码       │ 我当前要做的决策             │ 决策后果        │
└──────────────┴──────────────────────────┴──────────────┘
```

## 5.2 左侧：我手里有什么

左侧只放帮助玩家做决策的信息。

示例：

```text
【我的身份】
浙江总督

【我的当前目标】
稳住浙江
压住巡抚
防止皇帝疑我拖延
保住海防军心

【我的资源】
调粮权
密奏权
节制巡抚
海防军报
幕僚 4 人

【我的筹码】
半页田契暗账
县令密信
巡抚越级上奏传闻
```

左侧回答的问题是：

> **我能拿什么去做决策？**

## 5.3 中间：真正的游戏现场

中间不是系统日志，也不是事件卡堆叠。

中间是：

> **剧情消息流 + 当前决策区。**

玩家在这里看到：

```text
系统大势
其他角色行动
AI 角色反应
私密情报
待决策事件
自己的决策结果
```

中间回答的问题是：

> **现在局里发生了什么？我该如何应对？**

## 5.4 右侧：我做完之后世界怎么变了

右侧显示状态反馈。

示例：

```text
【世界状态】
国库银：42
民心：55
粮价：72
改桑进度：58
皇帝信任：43

【人物关系】
浙江巡抚：敌意 +3
清流县令：信任 -1
江南商会：观望
司礼监：警惕 +2

【最新变化】
你的密奏削弱了巡抚抢功效果。
但内阁开始怀疑你越级自保。

【当前风险】
巡抚反咬：高
粮价失控：中
县令公开暗账：中
```

右侧回答的问题是：

> **我的选择让这个世界发生了什么变化？**

# 6. 剧情消息流怎么玩

## 6.1 消息不是日志，而是剧情事件

错误写法：

```text
浙江巡抚选择了【越级上奏】。
```

正确写法：

```text
【第 3 天 · 午后】

驿站快马离开杭州府。

有人看见浙江巡抚的幕僚亲自护送一封急奏北上。奏中称：浙江改桑已有成效，只待朝廷嘉奖，十日内便可见第一批银。

你的幕僚低声提醒：

“大人，若这封奏疏先到内阁，巡抚便是功臣；若之后民怨爆发，您这个总督就是压不住局的人。”

你现在必须判断：是压下去，还是顺势利用？
```

## 6.2 消息类型

剧情消息分为 6 类。

### 1. 系统叙事消息

用于推进大势。

```text
【第 3 天 · 上午】

改桑令已下三日，浙江各县执行不一。米价连涨，民间怨声渐起。京师催银的诏令再次传来。
```

作用：

```text
提供背景
制造氛围
推动当天主题
```

### 2. 其他玩家行动消息

其他玩家或 AI 角色的行动，必须被转译成剧情。

```text
【浙江巡抚 · 行动】

巡抚刘瑾将改桑初成的奏疏送往京师。奏中只写桑田进展，不提粮价与民怨。

此举若成功，将提高巡抚在内阁中的声望，也会削弱你的统筹权威。
```

作用：

```text
告诉你别人动了
告诉你对方动作的目的
告诉你这件事如何影响你
```

### 3. AI 角色反应消息

AI 角色不会主动抢戏，但会根据局势反应。

```text
【清流县令 · 密信】

县令递来一封私信：

“粮价再涨，民恐不堪。另，巡抚与商会之间似有旧约，尚未取得实据。”
```

作用：

```text
提供可利用信息
制造插手点
增强“只有我知道”的权谋感
```

### 4. 私密情报消息

只有当前玩家能看见。

```text
【密报】

你的幕僚查到，江南商会昨夜派人入织造局，疑似寻求司礼监保护。
```

作用：

```text
制造信息不对称
让玩家判断是否公开、利用、隐瞒
```

### 5. 待决策消息

明确告诉玩家现在必须判断。

```text
【待你决断】

巡抚奏疏已经送出。
如果你不处理，内阁可能只听到巡抚一面之词。

你要如何应对？
```

作用：

```text
触发玩家决策
防止用户只是读剧情
```

### 6. 结果反馈消息

玩家决策后出现。

```text
【你的决定已生效】

你没有截留巡抚奏疏，而是追加一封密奏入京。

奏中写道：

“浙江可改，然不可躁进。臣非不愿速成，实恐桑田未成而民心先裂。”

结果：
皇帝信任 +4
内阁疑心 +2
巡抚抢功效果被部分抵消
司礼监开始关注浙江奏报差异
```

作用：

```text
让玩家看到反馈
让右侧状态变化有依据
制造下一条剧情钩子
```

# 7. 玩家如何做决策

## 7.1 每条关键消息给 3 个选项 + 自定义

标准结构：

```text
【你要如何应对？】

A. 截留奏疏
派人追上驿站，暂扣巡抚奏疏。
可能收益：阻止巡抚抢功。
可能风险：巡抚反咬你压制国策。

B. 追加密奏
不阻止奏疏，但另写一封密奏给皇帝。
可能收益：保留解释权。
可能风险：内阁怀疑你越级自保。

C. 放任巡抚
让他继续抢功，等待他与商会绑定更深。
可能收益：未来可一并清算。
可能风险：巡抚短期声望上升。

D. 自定义决策
输入你的处理方式。
```

## 7.2 玩家可以怎么决策

常见决策类型：

| 类型 | 含义 | 示例 |
|---|---|---|
| 压制 | 直接阻止对方动作 | 截留巡抚奏疏 |
| 密奏 | 越过常规渠道给上级留下说法 | 追加密奏给皇帝 |
| 放任 | 暂时不管，让对方继续暴露 | 让巡抚继续抢功 |
| 交易 | 用利益换对方配合 | 以减税换商会放粮 |
| 威慑 | 用权力或把柄逼对方让步 | 暗示商会账册会被查 |
| 借刀 | 利用别人替自己出手 | 让县令查巡抚暗账 |
| 留痕 | 留下未来自保证据 | 提前写下“不可躁进”公文 |
| 切割 | 失败时划清责任 | 把强征责任推给巡抚下属 |
| 公开 | 把私密线索变成公共压力 | 公开半页田契暗账 |
| 隐瞒 | 暂时保留信息 | 不透露海防真实军情 |

## 7.3 自定义决策

玩家可以自己写：

```text
我不直接拦截奏疏，而是派幕僚暗中抄录一份，确认巡抚到底在奏中隐瞒了什么。同时，我以总督府名义向内阁递一封平稳奏报，不正面反驳巡抚，只强调浙江粮价未稳，不宜急催。
```

AI 需要判断：

```text
是否符合角色身份
是否符合时代背景
是否拥有相关资源
是否越权
是否破坏当天主线
是否会导致无限发散
```

如果不合法，AI 给出改写建议。

示例：

```text
你不能直接处死巡抚，因为你没有这个合法权力。
更合理的做法是：先掌握巡抚与商会暗账，再以“执行过激、扰动民心”为名奏请问责。
```

# 8. 计策链：不是选动作，而是改变命运

## 8.1 动作不是权谋

低级动作：

```text
护官：保护县令，让他查账。
```

真正的权谋应该展开成：

```text
表面理由：
县令熟悉民情，暂时不宜调离。

真实目的：
利用县令查巡抚和商会的暗账。

利用对象：
县令的清名、百姓的信任、巡抚的急躁、商会的心虚。

预期效果：
让县令成为一把刀，逼巡抚和商会露出破绽。

隐藏风险：
县令如果失控，可能引发民变；巡抚如果反咬，可能说总督纵容抗旨。

后手：
若县令查出证据，则总督上奏“地方执行过激”；
若县令失败，则总督切割，说只是命其安抚民情。
```

## 8.2 每个计策至少有 6 层

```text
1. 目标：我想改变什么命运？
2. 障碍：谁挡路？
3. 筹码：我手里有什么？
4. 人物：我利用谁、压制谁、保护谁？
5. 行动：我表面怎么说，私下怎么做？
6. 后手：失败时怎么自保？
```

## 8.3 玩家可以选择简单选项，也可以写复杂计策

MVP 中，系统给普通用户 3 个选项，降低门槛。

高级玩家可以使用自定义输入，把自己的计策写出来。

AI 会把自定义内容转译成结构化决策：

```text
公开行动
私密行动
真实意图
利用对象
风险
后手
```

# 9. 命运下注机制

## 9.1 每个角色都有四层目标

每个角色都不是简单“完成任务”。

角色应有：

```text
公开目标：别人以为你要做什么
生存目标：你最低不能失去什么
晋升目标：你真正想得到什么
赌命目标：你愿意冒巨大风险换什么
```

示例：浙江巡抚

```text
公开目标：
完成改稻为桑，替朝廷筹银。

生存目标：
不能让官商低价夺田的证据暴露。

晋升目标：
把第一批银子和桑田数字做漂亮，调入京师。

赌命目标：
牺牲县令和部分商人，把所有脏事推给他们，自己成为“能办事的重臣”。
```

## 9.2 每个角色都有五档下场

```text
大胜：升官发财 / 得宠 / 掌权 / 名声大振
小胜：保住位置 / 得到部分利益 / 有后续机会
平局：活下来，但错失机会
小败：罢官 / 失势 / 破财 / 被调离
大败：下狱 / 抄家 / 流放 / 处死
```

玩家每一步都在改变自己的下场。

## 9.3 成功也会欠债

每个成功决策都可能产生“命运债”。

示例：

```text
你利用县令查账。
短期收益：获得巡抚与商会暗账。
未来风险：县令可能发现你只是利用他，不一定真要护民。

你逼商会放粮。
短期收益：粮价下降，民心回升。
未来风险：商会可能转投司礼监，寻求更强保护。
```

这让游戏更像真实世界。

# 10. AI 角色怎么玩

## 10.1 AI 角色不是主动玩家

单人模式中，其他角色由 AI 机器人扮演。

但 AI 机器人不是“自己玩游戏、主动抢戏”的 AI。

它们是：

> **局势参与者 / 反应角色 / 压力来源 / 交易对象。**

## 10.2 AI 角色什么时候行动

AI 角色只在三种情况下行动：

```text
1. 玩家影响到它
2. 主线事件推着它
3. 它的风险或机会达到阈值
```

示例：

```text
粮价 > 70
→ 清流县令请求开仓

皇帝疑心 > 75
→ 司礼监入浙探查

巡抚声望被压制
→ 巡抚越级上奏

商会清算风险过高
→ 商会投靠司礼监
```

## 10.3 AI 角色行动如何显示

AI 角色行动也不能显示成系统日志。

错误：

```text
商会 AI 选择了【投靠司礼监】。
```

正确：

```text
【密报】

你的幕僚查到，江南商会昨夜派人入织造局。那人带着两只封好的银匣，出门时只说了一句：“这条路，不能只靠巡抚府。”

如果消息属实，商会正在寻找比你更硬的保护伞。
```

# 11. 多人怎么玩

## 11.1 多人不要求同时在线

多人模式不要求所有人同时操作。

每个玩家在自己的视角中看到剧情消息流。

某个玩家不在线时，角色按默认惯性推进。

示例：

```text
巡抚玩家未上线
→ 巡抚默认继续推进改桑，保持向内阁报功倾向
```

这样不会因为一个玩家不在线导致卡局。

## 11.2 一个玩家的决策如何影响另一个玩家

例如总督玩家选择“追加密奏”。

总督看到：

```text
【你的决定已生效】

你没有截留巡抚奏疏，而是让幕僚连夜起草密奏。

奏中写道：
“浙江可改，然不可躁进。臣非不愿速成，实恐桑田未成而民心先裂。”

结果：
皇帝信任 +4
内阁疑心 +2
巡抚抢功效果被部分抵消
```

巡抚玩家看到的不是：

```text
总督选择了追加密奏。
```

而是：

```text
【京师回声】

你派出的急奏已经抵达通政司。

但你很快听说，浙江总督府也有一封密奏入京。内容未明，只知道其中提到了“民心”“粮价”“不可躁进”。

你的幕僚提醒：

“大人，总督没有拦你，但他在给自己留后手。若日后出事，他可能把你说成操切误国。”

【你要如何应对？】

A. 继续报功，抢在总督前面坐实功劳
B. 私下联络内阁，让内阁先支持你
C. 暂缓推进，避免被总督抓住把柄
D. 自定义决策
```

这就是多人互动的核心：

> **我的选择会变成别人的剧情压力。**

## 13.3 第 3 天示例

### 剧情消息

```text

```

### 决策选项

```text
A. 截留奏疏

可能风险：巡抚反咬你压制国策。

B. 追加密奏

C. 放任巡抚
让他继续抢功，等待他与商会绑定更深。
可能收益：未来可一并清算。
可能风险：巡抚短期声望上升。

D. 自定义决策
输入你的处理方式。
```

### 玩家选择 B 后的结果

你没有截留巡抚奏疏，而是让幕僚连夜起草密奏。

# 14. 日终回响怎么玩

每天结束时，系统必须给日终回响。

日终回响不是简单总结，而是告诉玩家：

```text
今天你做了什么
这件事改变了什么
谁开始怀疑你
谁开始依赖你
哪些风险留到了明天
```

示例：

```text
【第 3 天 · 日终回响】

你没有阻止巡抚急奏，而是另写密奏入京。

这让你保住了解释权，也让巡抚意识到你并非完全被动。

今日变化：
- 巡抚抢功效果被削弱
- 皇帝对你“谨慎持重”的印象略有提升
- 内阁开始怀疑你是否绕过正常奏报体系自保
- 司礼监注意到浙江奏报前后不一

明日风险：
- 粮价若继续上涨，县令可能请求开仓
- 巡抚可能反咬你拖延国策
- 商会可能借机投靠司礼监
```

# 15. 游戏如何结束

## 15.1 固定天数后进入裁决

例如标准 7 天局：

```text
第 7 天进入最终裁决
```

玩家不能无限继续拖延。

## 15.2 结局由什么决定

结局由以下因素决定：

```text
世界变量
角色变量
人物关系
关键决策
把柄流向
隐藏暗线
命运债
谁背锅
谁得利
谁保命
谁被清算
```

## 15.3 结局分两层

### 1. 全局结局

```text
本局结局：商人救国，内廷得利

改桑没有彻底失败。
国库得到第一笔银子。
但江南丝路从此被商会与司礼监共同控制。
内阁保住体面，浙江民心留下裂痕。
```

### 2. 个人结局

```text
你的最终下场：小胜

皇帝认为你尚可用，但不可不防。

关键三手：
1. 追加密奏，保留解释权
2. 逼商会平粮，暂稳民心
3. 放任巡抚抢功，最后借暗账反制

命运债：
你利用了县令，却没有真正保住他。
```

# 16. 如何防止游戏变成无限聊天

必须遵守以下硬规则：

```text
1. 每局有固定天数。
2. 每天有固定主线压力。
3. 每天消息数量有限。
4. 每次决策最多生成：
   - 1 条自己的结果消息
   - 1 条影响他人的剧情消息
   - 1 条隐藏暗线变化
5. 每条消息必须推动局势。
6. 每天必须进入日终回响。
7. 最后一天必须裁决。
8. 不允许无限追问。
9. 不允许所有角色自由闲聊。
10. 不允许 AI 无边界扩写。
```

判断一条消息是否应该出现：

```text
它是否带来新压力？
它是否提供新机会？
它是否暴露新风险？
它是否改变人物关系？
它是否产生决策点？
它是否影响最终结局？
```

如果都没有，就不应该进入主消息流。

# 18. 玩家最终应该感受到什么

玩家玩完一局后，应该感受到：

```text
我不是在读 AI 写的故事。
我是一个局中人。
别人做的事会推到我面前。
我必须判断要不要回应。
我的每个决策都有收益和风险。
我可以选择保命，也可以搏一把。
我可以影响别人，也会被别人影响。
最后我的下场，是我一步步推出来的。
```

如果玩家只是感觉：

```text
我看了一篇 AI 小说
我点了几个按钮
```

那玩法失败。

如果玩家感觉：

```text
我想再开一局，试试看换一种决策会不会有不同下场。
```

那玩法成立。

# 19. 给设计和开发的一句话

> **中间是游戏现场，不是信息面板。**

所有设计和开发都要围绕这个原则：

```text
中间：剧情消息流 + 当前决策
左边：我的身份、资源和筹码
右边：决策后的世界变化
```

任何不服务这个结构的复杂 UI，都应该先砍掉。

# 20. 最终总结

AI 多人局的玩法不是“AI 写故事”，而是：

> **AI 把一个局推到你面前，你用角色的权力、信息、筹码和判断做决策，AI 再把你的决策变成新的局势。**

最终闭环是：

```text
剧情压力
↓
玩家决策
↓
局势变化
↓
他人反应
↓
新剧情压力
↓
最终裁决
```

这就是这个游戏应该怎么玩。
