# AI 多人局｜MVP 用户试玩与数据验证方案 v1.2

> 文档类型：用户研究 / 产品验证 / 埋点与决策标准  
> 适用范围：《桑田诏》Web 单人 MVP  
> 核心问题：**用户是否因为自己的选择产生可见且可回溯的影响，而愿意持续玩到第 7 天并重开另一条路线？**

---

## 0. v1.2 修订说明

本版把试玩重点从“消息卡与可见因果卡”调整为“持续叙事流”。

试玩必须验证：

```text
用户是否把中间区域理解为正在发生的故事
用户是否理解 A/B/C 是建议而不是标准答案
用户是否会编辑建议或完全自定义
用户是否能在没有收益/风险预告时作出判断
用户是否理解“本次变化”来自刚才的决定
用户是否会使用默认隐藏的局势记录回看
关键事件弹窗是否有效但不过度打断
右侧主动谋划是否自然进入中间叙事流
```

主线决策、回应和主动谋划的输入均以 200 字为上限。
## 1. 试玩不是验收 UI

工程完成只能证明系统按设计运行。试玩要验证的是：

```text
玩家是否理解自己在“局”里做决策
玩家是否记得关键选择
玩家是否能解释后果来自哪一步
玩家是否感受到 AI 角色有自己的算盘
玩家是否愿意继续下一天
玩家是否愿意重玩另一条路线
```

如果用户最终只能说“剧情写得不错”，但不能指出哪一步帮助或伤害了自己，核心产品假设失败。

## 2. 关键假设

| 编号 | 假设 |
|---|---|
| H1 | 玩家在 3 分钟内能理解身份、目标和当前压力 |
| H2 | 玩家会把 A/B/C 当作建议而不是标准答案 |
| H3 | 玩家能理解故事后的“本次变化”与刚才决定之间的关系 |
| H4 | 第 1—3 天形成的伏笔能在第 4—6 天带来惊喜或压力 |
| H5 | AI 角色行为看起来来自自身利益，而不是剧情工具 |
| H6 | 第 7 天结局能让玩家认同“这是我一步步造成的” |
| H7 | 完成一局后，部分玩家愿意换路线重玩 |
| H8 | 玩家能区分主线决策和主动谋划 |
| H9 | 主动谋划让玩家感到提前布局，而不是第二套选择题 |
| H10 | 玩家能指出至少一次谋划如何改变线索、关系、风险或后续事件 |
| H11 | 连续叙事流比分页和消息后台更沉浸，且不会让玩家迷失当前位置 |
| H12 | 不展示收益/风险后，玩家仍能依靠故事、身份、资源和局势作判断 |
| H13 | 局势记录默认隐藏不会影响理解，但能在需要时提供有效回看 |
| H14 | 关键事件弹窗能让玩家意识到“别人影响了我”，同时不会造成频繁打断 |
## 3. 样本

第一轮：12—20 人。

建议结构：

```text
6—8 人：策略、权谋、历史、桌游用户
4—6 人：互动叙事、剧本杀、文字游戏用户
2—4 人：普通 AI 产品用户
```

尽量覆盖：

- 没听过产品的人；
- 熟悉明代题材与不熟悉题材的人；
- 喜欢自由输入与偏好系统选项的人。

内部开发者不计入正式样本。

## 4. 试玩方式

### 4.1 单次时长

```text
任务说明：3 分钟
完整试玩：30—45 分钟
访谈：15—20 分钟
总时长：约 50—65 分钟
```

### 4.2 主持要求

主持人只做：

- 说明“你是浙江总督，目标是七天内稳局并保住自己”；
- 提醒中间会持续出现故事、建议与变化；
- 提醒可以点击建议后继续修改，也可以直接写自己的决定；
- 首次进入右侧时只说明“每天有 2 次谋划机会，可以主动找人、调查或使用筹码”；
- 用户找不到回看入口时，只提醒“中间上方有局势记录”；
- 记录卡点和口头反应。

主持人不得解释：

- 哪个建议更优；
- 任何收益、风险或成功概率；
- 隐藏触发条件；
- 角色真实目标；
- 某一步未来会如何反噬；
- 是否应该使用谋划；
- 关键事件幕后是谁。
### 4.3 Think-aloud

邀请用户在关键决策前说出：

```text
你认为谁在做什么？
你最担心什么？
为什么选这一项？
你预期谁会受影响？
```

不强迫持续讲话，以免破坏沉浸感。

## 6. 核心指标定义

### 6.1 漏斗

#### 选角到开局转化率

```text
run_created / role_select_viewed
```

#### 第 1 天到第 3 天留存

```text
到达 day=3 的 run / 完成第 1 个决策的 run
```

#### 第 3 天到第 7 天完成率

```text
ending_viewed / 到达 day=3 的 run
```

#### 整局完成率

```text
ending_viewed / run_created
```

#### 日终继续率

```text
advance_day_clicked / day_completed
```

### 6.2 行为指标

```text
每段故事平均阅读时间
叙事流向上回看次数
新内容提示点击率
每次决策选择时间 P50/P90
AI 建议点击率
点击建议后的编辑率
完全自定义决策使用率
200 字上限触发率
推演等待放弃率
“本次变化”可见时长
局势记录打开率
局势记录打开后的返回率
关键事件立即处理率
关键事件稍后处理率
关键事件延后后完成率
主动谋划使用率
每局平均谋划次数
人物交谈 / 调查 / 使用筹码 / 自拟谋划分布
谋划机会放弃率
主线决策与主动谋划混淆率
ActionGuard 拒绝率
结局页停留时间
结局分享意愿
重开一局率
```
### 6.3 理解指标

试玩后要求用户不看页面回答：

1. 你最记得哪一个决定？
2. 这个决定之后，故事发生了什么？
3. 哪一项数值或关系发生了变化？
4. 后来它有没有帮助或伤害你？
5. 哪个角色因为你的行为改变了态度？
6. 你的最终结局为什么是这个结果？
7. 主线决策和主动谋划有什么区别？
8. 你做过哪一次谋划？它后来改变了什么？
9. 局势记录是做什么的？
10. 关键事件弹窗表示什么？

前 6 题每题 0—2 分：

```text
0：完全说不出或明显错误
1：能说出大概，但因果不清
2：能指出具体决定、故事结果和变化
```

前 6 题用于叙事因果理解评分，总分 12 分；其余题单独记录玩法理解。
## 7. 第一轮目标阈值

以下是 MVP 假设阈值，不是行业事实。试玩后可调整。

| 指标 | 继续优化阈值 | 强信号 |
|---|---:|---:|
| 选角→开局 | ≥ 70% | ≥ 85% |
| 到达第 3 天 | ≥ 65% | ≥ 80% |
| 第 3 天→结局 | ≥ 50% | ≥ 65% |
| 整局完成率 | ≥ 40% | ≥ 55% |
| 日终继续率 | ≥ 70% | ≥ 85% |
| 自定义决策使用率 | ≥ 15% | ≥ 30% |
| 至少使用 1 次主动谋划 | ≥ 50% 用户 | ≥ 70% 用户 |
| 平均每局谋划次数 | ≥ 3 次 | ≥ 6 次 |
| 能正确区分主线决策与主动谋划 | ≥ 70% 用户 | ≥ 85% 用户 |
| 能说出一次谋划的后续影响 | ≥ 50% 用户 | ≥ 70% 用户 |
| 叙事因果理解得分 ≥ 8/12 | ≥ 60% 用户 | ≥ 75% 用户 |
| 愿意重玩 | ≥ 35% | ≥ 50% |
| 愿意推荐/分享结局 | ≥ 25% | ≥ 40% |

单局时长超过 45 分钟且完成率低，需要缩短文本或提高节奏，不应先增加更多内容。

## 8. 定性访谈问题

### 8.1 入局感

- 你什么时候开始觉得自己是浙江总督，而不是在答题？
- 哪条消息让你第一次感到别人正在出招？
- 左侧身份、资源、目标是否足以帮助你判断？

### 8.2 决策感

- A/B/C 看起来是 AI 建议、策略方向，还是标准答案？
- 不展示收益和风险后，你是否知道该依据什么做判断？
- 点击建议后继续修改是否自然？
- 你是否愿意完全自己写？
- 输入框一直显示是否清楚？
- 哪次选择最难？为什么？
- “AI 正在推演局势……”是否足够，还是你希望看到更多过程？
### 8.3 主动谋划

- 你是否理解右侧“谋划”与中间主线决策的区别？
- 你什么时候第一次产生“我可以提前布局”的感觉？
- 人物交谈、调查、使用筹码和自拟谋划中，哪个最自然？
- 每天 2 次谋划机会是太少、合适还是太多？
- 未使用的谋划机会是否让你有损失感或压力？
- 谋划结果进入消息流后，你是否能看出它改变了什么？
- 右侧是否像第二套选择题，还是像主动布局工具？

### 8.4 因果感

- 结果故事结束后，你是否注意到“本次变化”？
- 变化与刚才决定之间是否清楚？
- 哪一步后来帮助了你？
- 哪一步当时有利、后来变成风险？
- 你是否需要打开局势记录才能回忆？
- 局势记录是帮助理解，还是像后台日志？
- 数值变化是否过多、过少或过于突兀？
### 8.5 角色真实性

- 巡抚、县令、商会、司礼监中，谁最像有自己的算盘？
- 哪个角色最像工具人？
- 关键事件弹窗是否让你感觉别人真的改变了你的处境？
- 弹窗出现频率是太少、合适还是太多？
- “稍后处理”后，你是否知道事件仍未解决？
- 是否出现角色知道了不该知道的信息？
- 某个角色的反应是否让你意外但合理？
### 8.6 结局与重玩

- 你认同最终裁决吗？
- 哪句话让你觉得结局来自自己的选择？
- 你愿意重玩吗？下一次最想改哪一步？
- 你希望下一局是历史、商战、职场还是悬疑？

## 10. 产品决策规则

### 10.1 可以进入下一阶段

同时满足：

```text
整局完成率达到继续优化阈值
多数用户能说出具体因果
至少一个延迟后果被用户主动提及
多数用户理解主动谋划并能说出其影响
AI 角色真实性评价明显高于“工具人”
重玩率达到 35% 左右或以上
无严重状态丢失、结局不一致、信息泄露
```

下一阶段优先：

- 优化剧情节奏；
- 增加更多自定义决策质量；
- 优化角色反应与因果回溯；
- 再制作第二个故事局。

### 10.2 暂停扩展，继续修核心

出现任一：

```text
多数用户只说“剧情不错”但说不出因果
第 3 天前大量流失
故事后的变化摘要被认为突兀、啰嗦或看不懂
角色反应明显像系统工具
结局被认为随机或不公平
自定义决策经常被拒或与预设无差异
用户把主动谋划理解为第二套主线选择题
多数用户完全不使用谋划或看不出谋划结果
谋划机会造成明显“必须用完”的负担
```

此时不要开发真人多人、支付、UGC 或更多剧本。

### 10.3 核心假设失败

若两轮优化后仍出现：

```text
整局完成率持续低于 30%
因果理解显著低于 50%
几乎无人愿意重玩
用户更愿意直接看故事而不是做决策
```

应重新评估消息密度、决策频率、因果展示和故事题材，而不是继续堆功能。

## 11. 两轮测试计划

### 第一轮：可理解性

```text
样本：6—8 人
目标：找到流程卡点、文本长度、因果理解和角色真实性问题
版本：staging 固定版本
```

修正后进入第二轮。

### 第二轮：行为验证

```text
样本：12—20 人
目标：测量漏斗、完成率、自定义使用、谋划使用、主线/谋划理解和重玩意愿
版本：冻结核心规则，仅允许修严重 bug
```

第二轮结束后形成：

```text
MVP 用户验证报告 v1.0
继续 / 调整 / 暂停结论
下一阶段功能优先级
```
