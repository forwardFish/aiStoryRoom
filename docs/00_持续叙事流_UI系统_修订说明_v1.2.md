# AI 多人局｜持续叙事流 UI 系统修订说明 v1.2

> 目标：把《AI 多人剧情推演：主游戏页持续叙事流 UI / 交互设计文档（MVP）》正式写入工程基线、状态机、API、配置、测试、部署、试玩、PRD、数据库、玩法和《桑田诏》剧本，避免只改视觉而没有后端闭环。

## 一、本次修订的核心结论

主游戏页不再以“消息卡片列表 + 独立结果页”为默认体验，而统一为：

```text
连续故事
→ AI 提供 3 个行动建议
→ 玩家选择建议或输入自己的方案
→ 提交
→ AI 正在推演局势……
→ 新故事自动追加
→ 可见变化自动追加
→ 如有新压力，继续出现下一次决策
```

页面骨架始终固定：

```text
顶部：时间、地点、次数与世界状态
左侧：身份、目标、资源、筹码与风险
中间：持续叙事时间线
右侧：主动谋划入口
```

只有中间区域切换或追加内容。

## 二、8 个正式 UI 状态

```text
UI 01 角色专属开场
UI 02 主线故事与决策
UI 03 AI 正在推演
UI 04 推演结果故事与变化
UI 05 局势记录
UI 06 关键事件弹窗
UI 07 他人影响故事与回应
UI 08 主动谋划
```

以上是 8 种页面状态，不是 8 个路由。正式游戏仍只有一个主页面：

```text
/game?runId=<runId>
```

## 三、统一交互规则

### 3.1 决策选项

前台只展示：

```text
A 行动名称
B 行动名称
C 行动名称
```

不得展示：

```text
收益
风险
成功率
数值预测
后果说明
```

策划配置中可以保留内部说明、状态补丁和风险标签，但公共 API 与前端投影不得返回这些内部字段。

### 3.2 自定义输入

主线决策、关键事件回应、主动谋划共用同一套输入逻辑：

- 点击 AI 建议后，将建议标题写入输入框；
- 玩家可以继续补充或改写；
- 玩家也可以不选建议，直接输入自己的方案；
- 主线决策和主动谋划最大 200 字；
- 最终提交的是玩家确认后的完整文本。

### 3.3 推演状态

前台只显示：

```text
你的决定
<玩家最终提交内容>

AI 正在推演局势……
```

不得展示模型、权重、判断步骤、概率、Token、技术日志或后端任务进度。

### 3.4 结果展示

推演完成后自动追加：

```text
结果故事
→ 本次变化
→ 下一次决策（如有）
```

不使用“查看变化”“继续阅读”“返回局势”“完成”等流程按钮。

### 3.5 局势记录

局势记录默认隐藏，只在中间区域展开，用于回看：

```text
我的决策
他人影响
世界变化
数值变化
```

它不是独立页面，不展示复杂因果图，关闭后必须恢复原叙事滚动位置。

### 3.6 关键事件

关键事件弹窗是唯一保留“立即处理 / 稍后处理”双按钮的流程。

MVP 中需要玩家回应的关键事件仍占用当日 2 次主线决策中的一个决策槽位，不额外增加单局 12 次主线决策和 AI 调用预算。

## 四、统一术语

前台：

```text
持续叙事时间线
主线决策
主动谋划
谋划机会
自拟谋划
提交决策
提交回应
提交谋划
局势记录
本次变化
```

代码与 API：

```text
narrativeEntry
entryType
activePrompt
promptKind
decision
maneuver
criticalEvent
changeSummary
situationRecord
```

继续禁止：

```text
AP
行动点
行动力
筹谋
筹谋机会
```

## 五、本次需要修改的文档

| 文档 | 修改原因 |
|---|---|
| 01 MVP 唯一工程基线 | 将持续叙事流与 8 个 UI 状态提升为最高工程约束 |
| 02 StoryRun 状态机与 API | 增加 activePrompt、关键事件延后、叙事投影与极简 resolving |
| 03 剧本配置 Schema | 区分前台标题与后台收益/风险，统一 200 字，增加关键事件展示配置 |
| 04 自动化验收矩阵 | 增加连续滚动、选项回填、无风险预览、局势记录和弹窗测试 |
| 05 部署、日志与 AI 成本 | 统一 200 字、极简推演文案、静默 fallback 与公共投影检查 |
| 06 用户试玩方案 | 从“因果卡理解”改为“故事 + 本次变化 + 局势记录”的理解验证 |
| 07 Web MVP PRD | 重写主游戏页、决策、结果与 8 个 UI 状态 |
| 08 数据库与存储架构 | 增加叙事投影、activePrompt、criticalEvent 与 defer 事件 |
| 09 游戏玩法说明 | 将默认体验从消息卡片流改为持续叙事时间线 |
| 10 《桑田诏》剧本 | 所有玩家可见选项只保留标题，收益/风险转为后台配置 |
| 主游戏页 UI 交互设计 | 修正路由、术语、MVP 单人边界及状态字段 |
