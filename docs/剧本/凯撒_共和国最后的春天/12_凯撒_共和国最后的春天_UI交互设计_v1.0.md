# 《凯撒：共和国最后的春天》Web UI / 交互设计文档 v1.0

> 文档类型：完整 UI 页面设计说明 / Codex 开发规格 / 9 张设计图统一依据  
> 适用产品：Many Worlds 海外 Web 版  
> 对应剧本：《凯撒：共和国最后的春天》完整故事局剧本 v1.0  
> 默认主控角色：马库斯·尤尼乌斯·布鲁图斯（Marcus Junius Brutus）  
> 页面结构：1 个角色选择页 + 1 个主游戏路由的 8 种页面状态  
> 设计画布：2048 × 1152，16:9  
> 核心原则：**这是 Many Worlds 中的另一个完整世界，不是重新设计一套新产品。保留嘉靖版本的产品骨架、交互逻辑、品牌语言与页面比例，只替换世界场景、人物、数据、图标和叙事语义。**

---

# 0. 本次设计的核心修正

此前出现的问题，不是单纯“文字没有换成凯撒”，而是没有把《凯撒：共和国最后的春天》当作一个完整的罗马世界来设计。

错误方向包括：

```text
只把郑坤彬替换成布鲁图斯；
只把杭州粮局替换成匿名信；
背景仍然是中国山水、宝塔、江南建筑；
数值仍然是银两、粮价、改革进度、皇帝信任；
人物动作仍然是劝文、策反、商议粮价；
页面有时变成棕色羊皮卷或黑暗战争游戏；
不同 UI 图的布局、数据、人物和风格互相不一致。
```

正确方向应当是：

```text
Many Worlds 平台品牌不变；
整体页面结构不变；
白色 + 淡紫蓝的轻盈视觉不变；
三栏式主游戏骨架不变；
持续叙事流不变；

但整个世界层必须彻底变成公元前 44 年的罗马：
罗马城市、元老院、宅邸、广场、神庙、柱廊、蜡板、卷轴、桂冠、鹰徽；
布鲁图斯、凯撒、卡西乌斯、安东尼、德西穆斯、西塞罗；
共和声望、凯撒信任、元老院合法性、军团忠诚、密谋暴露、内战风险；
第 1—7 幕，而不是第 1—7 天；
距离三月十五日，而不是距离御前裁决；
罗马裁决，而不是御前裁决。
```

一句话定义：

> **看上去必须仍然是同一个 Many Worlds 产品，但用户进入后必须立即知道：自己已经离开大明，进入了公元前 44 年的罗马。**

# 1. 产品页面的核心定义

## 1.1 页面不是“古罗马游戏网站”

不能把页面重新设计成：

```text
深色战争游戏；
棕黄色羊皮卷；
全屏罗马战场；
《全面战争》式复杂策略面板；
古罗马博物馆网页；
独立 SPQR 品牌网站。
```

Many Worlds 仍是平台品牌，凯撒只是平台中的一个世界。

因此必须保留：

```text
左上角 Many Worlds Logo；
轻白、淡紫蓝、低饱和度的品牌视觉；
固定顶部状态栏；
固定左侧角色栏；
固定右侧谋划栏；
中间持续叙事流；
紫蓝渐变的主操作按钮；
细边框、轻阴影、舒适留白。
```

## 1.2 主游戏体验

主流程保持：

```text
故事发生
  ↓
玩家看到角色视角下的压力
  ↓
AI 给出 3 个行动方向
  ↓
玩家选择或输入自己的决定
  ↓
AI 推演局势
  ↓
结果故事出现
  ↓
个人、他人、世界变化出现
  ↓
新故事继续生长
```

## 1.3 页面三区分工

```text
左侧：
我是谁、我相信什么、我掌握什么、我暴露了多少。

中间：
此刻罗马发生了什么、我如何回应、世界因此发生了什么变化。

右侧：
我还能接触谁、调查什么、使用什么筹码、主动推进什么计划。
```

# 2. 与《嘉靖财政危局》页面的继承关系

## 2.1 必须原样继承的“大方向”

| 页面层级 | 嘉靖版本 | 凯撒版本要求 |
|---|---|---|
| 平台品牌 | Many Worlds | 完全不变 |
| 页面比例 | 16:9 三栏布局 | 完全不变 |
| 顶部结构 | 世界位置、日期、倒计时、决策、谋划、历史、设置 | 完全不变，只换内容 |
| 第二状态行 | 5 个核心世界指标 | 完全不变，只换罗马指标 |
| 左栏 | 身份、命运问题、目标、资源、筹码、风险 | 完全不变，只换罗马内容 |
| 中间 | 持续叙事流 | 完全不变，只换故事与背景 |
| 右栏 | 谋划中枢 | 完全不变，只换人物、目标、筹码 |
| 主按钮 | 紫蓝渐变 | 完全不变 |
| 卡片样式 | 白底、淡紫描边、轻阴影 | 完全不变 |
| 背景插画 | 中国水墨山水 | 替换为淡紫蓝罗马线描水彩 |

## 2.2 必须彻底替换的“世界层”

| 嘉靖世界 | 凯撒世界 |
|---|---|
| 杭州总督府 · 内厅 | 罗马 · 布鲁图斯宅邸 / 元老院 / 广场 |
| 第 3 天 · 午后 | 第 3 幕 · 清晨 |
| 距离御前裁决 4 天 | 距离三月十五日 4 天 |
| 国库银 | 凯撒权威 / 城市秩序等罗马指标 |
| 民心 | 凯撒民望 / 平民态度 |
| 粮价 | 不再出现 |
| 改革进度 | 密谋凝聚 / 协议进度 |
| 皇帝信任 | 凯撒信任，只出现在个人资源区 |
| 巡抚、商会、司礼监 | 凯撒、卡西乌斯、安东尼、西塞罗等 |
| 密报、奏疏、账本 | 匿名信、蜡板、元老名单、凯撒法令、警告信 |
| 进入下一天 | 进入下一幕 |
| 御前裁决 | 罗马裁决 |

# 4. 整体视觉语言

## 4.1 必须保持的品牌基调

当前嘉靖页面的真正优势是：

```text
明亮
克制
有叙事感
不压迫
不像传统战争游戏
既有历史氛围，又有现代产品可用性
```

凯撒版本继续使用：

```text
大面积柔和白色
极浅灰紫分区
淡紫蓝描边
深靛蓝正文
紫蓝渐变主按钮
小面积青绿表示正向状态
小面积朱红表示风险
轻微阴影
中等圆角
大量留白
```

## 4.2 罗马世界色彩

主品牌色不变，只加入少量罗马辅助色。

| 用途 | 建议色值 | 说明 |
|---|---|---|
| Many Worlds 主紫 | `#5C43F5` | 主按钮、选中态、交互重点 |
| 深靛蓝 | `#15214A` | 标题、正文、数字 |
| 淡紫描边 | `#DDD9FA` | 卡片边框、分隔线 |
| 页面白 | `#FCFCFF` | 主背景 |
| 罗马象牙白 | `#F8F5EE` | 场景插画和局部底色 |
| 青铜金 | `#B58A4A` | 硬币、桂冠、卷轴筹码的小图标 |
| 共和国红 | `#A6403C` | 高风险、关键事件，不作为大面积主色 |
| 正向绿 | `#2E8A55` | 谋划次数、正向变化 |
| 雾蓝紫 | `#E9EAFE` | 场景线稿、背景山墙、柱廊 |

## 4.3 明确禁止

```text
不要整页棕黄羊皮卷；
不要黑色、暗红色战争游戏；
不要强烈火焰、血色、战场；
不要让金色成为主按钮颜色；
不要替换 Many Worlds Logo；
不要出现中国山水、宝塔、江南船只；
不要出现中世纪欧洲城堡；
不要出现帝国时期穹顶、巴洛克城市或现代罗马建筑；
不要过度使用 SPQR 标识；
不要把页面做成博物馆展板。
```

## 4.4 罗马视觉符号

推荐使用：

```text
罗马柱廊
神庙三角楣
元老院台阶
古罗马广场
布鲁图斯宅邸中庭
祖先半身像
桂冠
蜡板
卷轴
印戒
罗马鹰徽
秤与法令牌
石雕与喷泉
```

不建议把束棒作为主要品牌符号，避免形成现代政治联想。

# 5. 场景背景系统

背景不能只做一张“泛罗马城市图”重复使用。

每一幕有自己的主要场景资产，但必须使用同一种画法：

```text
淡紫蓝线描 + 极低饱和水彩
主要集中在中间区域下半部
透明度约 15%—30%
不影响正文阅读
有前景与远景，但不抢文字
```

## 5.1 七幕场景对应

| 幕次 | 主要场景 | 视觉内容 |
|---:|---|---|
| 第 1 幕 王冠 | 罗马广场 | 节庆人群、神庙、雕像、远处高台 |
| 第 2 幕 终身独裁者 | 元老院 / 议事厅 | 柱廊、议席、蜡板、法令卷轴 |
| 第 3 幕 写给布鲁图斯的信 | 布鲁图斯宅邸 | 中庭、祖先像、书案、匿名信、晨光 |
| 第 4 幕 谁必须一起倒下 | 卡西乌斯宅邸 / 夜间柱廊 | 油灯、阴影、关闭的门、秘密会面 |
| 第 5 幕 三月十五日前夜 | 凯撒宅邸 | 暴雨、油灯、门外信使、室内雕像 |
| 第 6 幕 元老院之日 | 庞培剧场议事厅 | 高大柱廊、议员、会场入口、广场 |
| 第 7 幕 罗马裁决 | 卡庇托利山 / 广场 | 黎明、群众、元老、远处罗马城 |

## 5.2 第 3 幕统一设计图背景

本轮 8 张主游戏 UI 图统一采用：

```text
布鲁图斯宅邸中庭
左下或右下出现半透明罗马柱廊
远处是神庙与雕像轮廓
桌面可隐约出现蜡板与信件
不出现人物大场景
```

UI 06 弹窗后仍能辨认这是布鲁图斯宅邸，而不是中国内厅。

# 6. 字体与排版

## 6.1 中文设计稿

建议：

```text
页面标题：思源宋体 / Noto Serif SC SemiBold
正文：思源宋体 / Noto Serif SC Regular
数字与状态：Inter / 思源黑体
拉丁人物名：Cinzel 或其他古典衬线体
```

## 6.2 海外英文版

```text
世界与章节标题：Cinzel / Cormorant Garamond
正文：Source Serif 4 / Merriweather
按钮、状态与数字：Inter
```

## 6.3 排版原则

```text
正文宽度控制在中间区的 62%—72%；
单行英文正文不超过 78 个字符；
中文行距 1.8 左右；
英文行距 1.55—1.7；
章节标题居中；
地点标题左对齐；
决策区宽度略窄于正文；
背景插画不得穿过高对比文字区域。
```

# 25. 验收标准

## 25.1 一眼识别验收

用户不读正文时，也必须在 3 秒内看出：

```text
这是 Many Worlds；
这是古罗马；
我扮演布鲁图斯；
凯撒与元老院正在发生冲突；
三月十五日正在逼近。
```

## 25.2 骨架一致性验收

- [ ] Many Worlds Logo、顶部、左栏、右栏与嘉靖版本保持同一产品结构。
- [ ] 页面不是重新设计的新游戏网站。
- [ ] 9 张图的比例、栏宽、高度、卡片样式一致。
- [ ] 主游戏只有中间区域变化。

## 25.3 罗马世界验收

- [ ] 不出现山水、宝塔、江南船只、明代官服。
- [ ] 地点、背景、图标、人物、资源和用语全部属于罗马世界。
- [ ] 不出现中世纪城堡和骑士。
- [ ] 不使用深色战争游戏视觉。
- [ ] 罗马元素清楚，但不压过 Many Worlds 品牌。

## 25.4 内容一致性验收

- [ ] 统一为第 3 幕、清晨、距离三月十五日 4 天。
- [ ] 世界指标固定为凯撒权威、元老院合法性、凯撒民望、军团忠诚、内战风险。
- [ ] 左侧固定为布鲁图斯身份与状态。
- [ ] 右侧固定为凯撒、卡西乌斯、西塞罗、波西亚等可接触人物。
- [ ] 9 张图不随意改变数字和角色身份。

## 25.5 交互验收

- [ ] 选项只显示 A/B/C 和行动标题。
- [ ] 不展示收益、风险、概率。
- [ ] 点击建议写入输入框。
- [ ] 推演完成后自动追加故事和变化。
- [ ] 局势记录只在中间展开。
- [ ] 关键事件可立即处理或稍后处理。
- [ ] 右侧动作进入中间主动谋划。
- [ ] “进入下一幕”在条件不满足时禁用。

## 25.6 海外版验收

- [ ] 英文人物名与按钮不会挤压布局。
- [ ] 英文正文阅读宽度合理。
- [ ] 不依赖中国官职、皇权或奏疏语义。
- [ ] 用户不懂罗马官制时，仍能理解人物关系和当前冲突。

# 27. 最终产品体验定义

玩家进入凯撒世界后，应该感受到：

```text
我仍然在使用熟悉的 Many Worlds；
但这个世界的建筑、人物、权力结构和风险都已经完全改变；
我不是在一个中国页面上阅读罗马文字；
我是在布鲁图斯的宅邸、元老院和罗马广场之间推动自己的命运；
我的身份、筹码和风险属于罗马；
我的决定会改变凯撒、卡西乌斯、安东尼和共和国；
三月十五日不是固定结局，而是一条正在被所有人共同改变的世界线。
```

最终定义：

> **《凯撒：共和国最后的春天》的 UI 不是嘉靖页面换皮，也不是另起炉灶。它必须在保持 Many Worlds 统一产品骨架的同时，建立一个从场景、人物、状态、语言到因果反馈都完整成立的古罗马世界。**
